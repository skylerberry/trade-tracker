# Screenshots

Drop screenshots of the live app here for Claude (or any future contributor) to read as visual context — what the working site actually looks like, not just what the code says it should look like.

## Conventions

- **`current-light.png`** — current state, light mode
- **`current-dark.png`** — current state, dark mode
- **`current-mobile.png`** — mobile/iOS view (optional)

Overwrite these in place rather than accumulating dated copies — they go stale fast and bloat the repo.

If you want to preserve a historical snapshot (e.g., before/after a redesign), use a dated filename: `2026-05-05-pre-add-to-position.png`.

## When to update

- After visible UI changes (new components, layout shifts, color tweaks)
- Before/after a redesign you want to remember
- When asking Claude to work on UI — refresh `current-*.png` first so the read matches reality
